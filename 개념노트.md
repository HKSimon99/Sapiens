# 📚 개념노트 — Zettelkasten 방식 Atomic Notes

> **루만의 원칙**: 양이 아니라 **연결**. 잘 연결된 영구 노트 10개 > 고립된 캡처 100개.
> **Feynman**: 자기 말로 쓰지 못하면 모르는 것.
> **5년 후 테스트**: 5년 후의 내가 읽어도 이해할 수 있게 쓰기.

---

## 🎯 Zettelkasten 4원칙

1. **Atomic** — 한 노트에 하나의 아이디어만. 더 쪼갤 수 있으면 쪼개기.
2. **Own Words** — 인용 금지. 모든 노트는 내 언어로.
3. **Linking** — 새 노트 작성 시 반드시 기존 노트와 연결 ([[#개념ID]] 형식).
4. **Permanent** — 출처와 분리해도 그 자체로 이해 가능해야 함.

---

## 🗂️ Domain 1 — 인지 아키텍처

### [#D1-001] Inversion (역전 사고)
- **도메인**: D1
- **태그**: #인지아키텍처 #멘탈모델 #의사결정
- **한 줄 정수**: 목표에 직접 달려가는 대신, 실패 경로를 먼저 열거해서 봉쇄한다

**내 말로** (Feynman 어린이 설명):
이루고자 하는 바에 더 가까워지기 위해, 그 과정에서 하면 안 되는 것들을 피하는 사고의 전환. "어떻게 성공할까?" 대신 "어떻게 반드시 실패할까?"를 먼저 묻는다.

**왜 중요한가**:
인간은 장기 성과에 불리한 존재다 — 욕구와 즉각적 보상에 끌리기 때문에, 목표로 가는 직선 경로보다 이탈 경로를 먼저 봉쇄하는 게 현실적이다. 또한 인간은 이득보다 손실에 2배 민감하므로(Loss Aversion), "잃지 않을 것들"을 정의할 때 실행력이 더 강해진다.

**반례·한계**:
- 실패 경로 열거가 지나치면 행동 자체를 마비시킬 수 있다 (분석 마비)
- 전례 없는 혁신 상황에서는 알려진 실패 경로가 없어 Inversion만으론 부족하다

**연결 (Linking)**:
- [[#D1-002]] First Principles — 둘 다 전제를 해체하는 접근. Inversion은 목표를, FP는 문제를 해체
- [[#D1-003]] 확증편향 — Inversion이 막아야 할 대표적 실패 경로 (학습 예정 Week 3)

**경영·일상 적용**:
- 주식: "어떻게 잃을까?" → 전액 한 종목 투입 / 확증편향으로 버티기 / 시장 역행 고집 → 이 세 가지를 안 하면 된다
- 경영: "어떻게 조직을 망칠까?" → 소통 차단, 책임 회피, 단기 성과 집착 → 봉쇄 목록이 곧 리더십 원칙

**파생 질문**:
- Inversion으로 뽑은 실패 목록이 서로 충돌할 때는 어떻게 우선순위를 정하나?
- 실패 경로를 모르는 완전히 새로운 영역에서는 어떤 도구를 써야 하나?

**메타**:
- 첫 학습일: 2026-04-29
- 출처: Sapiens 세션 1 / Munger, C. *Poor Charlie's Almanack*
- 복습: D+1 ☐ / D+3 ☐ / D+7 ☐ / D+14 ☐ / D+30 ☐ / D+90 ☐

### [#D1-002] First Principles (제1원리 사고)
- **도메인**: D1
- **태그**: #인지아키텍처 #멘탈모델 #의사결정
- **한 줄 정수**: "당연한 것"을 의심하고, 가장 기초적인 사실로 분해한 뒤 처음부터 다시 쌓는다

**내 말로** (Feynman 어린이 설명):
사실 검증(부정)의 과정을 거쳐 당연하다고 여겨지는 것들을 근본적인 단위부터 다시 생각하는 과정. 남들이 주는 정보를 그대로 가져다 쓰는 게 아니라, "왜 이게 당연한가?"부터 묻는다.

**왜 중요한가**:
유추 사고(남들이 했으니 나도)는 기존 한계를 그대로 물려받는다. First Principles는 기초 사실에서 출발하기 때문에 업계 상식을 깨는 해결책이 나온다. 머스크가 배터리 원가를 kg당 $600 → $80으로 재계산한 것처럼.

**반례·한계**:
- 모든 것을 First Principles로 분해하면 시간이 너무 걸린다 — 일상적 결정엔 유추 사고가 효율적
- 기초 사실 자체가 틀렸을 경우 잘못된 방향으로 쌓게 된다

**연결 (Linking)**:
- [[#D1-001]] Inversion — 둘 다 "당연한 것을 의심"하는 데서 시작. Inversion은 목표를, First Principles는 방법을 해체
- [[#D1-003]] 확증편향 — FP가 막아야 할 함정. "기초 사실"을 편향적으로 선택할 위험 (Week 3 학습 예정)

**경영·일상 적용**:
- 환율 분석: "원화 가치가 녹는다"를 금리 하나로 단정하지 않고 무역수지·물가·자본흐름·지정학까지 분해 → 복수의 압력이 충돌한다는 결론
- 창업: "이 업계에선 원래 이 마진율"이라는 전제를 분해 → 실제 원가 구조를 보면 다를 수 있다

**파생 질문**:
- 어떤 사실이 "기초"인지 어떻게 판단하는가? (무한 분해의 함정)
- First Principles와 Systems Thinking은 어떻게 연결되는가?

### [#D1-003] Second-Order Thinking (2차 사고)
- **도메인**: D1
- **태그**: #인지아키텍처 #멘탈모델 #의사결정
- **한 줄 정수**: "그래서 어떻게 될까?"를 최소 두 번 반복해서 보이지 않는 연쇄 파생효과까지 추적한다

**내 말로** (Feynman 어린이 설명):
단순한 두 현상의 관계만 보는 게 아니라, 그 이후에 일어날 수 있는 연쇄적인 파생효과들도 고려하는 사고방식. 1차 결과는 보이고, 2차 결과는 보이지 않는다 — 좋은 판단자는 보이지 않는 것을 함께 본다.

**왜 중요한가**:
대부분의 사람은 1차 결과(직접적·즉각적)만 보고 결정한다. 2차·3차 결과를 미리 그리면 예상치 못한 일에 대비할 수 있고, 결정 전에 더 많은 선택지가 생긴다. 미리 알고 결정하는 것과 모르고 결정하는 것은 다르다.

**반례·한계**:
- 연쇄를 너무 멀리 추적하면 불확실성이 너무 커져 오히려 판단이 마비될 수 있다
- 2차 결과를 잘못 예측하면 오히려 잘못된 확신을 가질 위험이 있다

**연결 (Linking)**:
- [[#D1-001]] Inversion — 같은 맹점을 다른 방향에서 잡는다. Inversion은 "무엇을 피할까", SOT는 "그러면 어떻게 될까"
- [[#D1-002]] First Principles — SOT의 2차 결과 분석에 FP의 분해가 필요할 때가 많다

**경영·일상 적용**:
- 취직 결정: 1차(취직) → 2차(CFA 시간 부족) → 3차(건강 악화). 미리 알았다면 CFA 일정 재조정, 운동 루틴 설계가 가능했다
- 정책: 대출 규제 → 집값 하락(1차) → 전셋값 폭등(2차) → 임차인 부담 증가(3차)

**파생 질문**:
- 몇 차까지 추적하는 게 실용적인가? 어디서 멈춰야 하나?
- SOT와 시나리오 플래닝은 어떻게 다른가?

**메타**:
- 첫 학습일: 2026-05-01
- 출처: Sapiens 세션 3 / Howard Marks / Bastiat (보이는 것과 보이지 않는 것)
- 복습: D+1 ☐ / D+3 ☐ / D+7 ☐ / D+14 ☐ / D+30 ☐ / D+90 ☐
