# 📚 개념노트 — Zettelkasten 방식 Atomic Notes

> **루만의 원칙**: 양이 아니라 **연결**. 잘 연결된 영구 노트 10개 > 고립된 캡처 100개.
> **Feynman**: 자기 말로 쓰지 못하면 모르는 것.
> **5년 후 테스트**: 5년 후의 내가 읽어도 이해할 수 있게 쓰기.

---

## 🎯 Zettelkasten 4원칙

1. **Atomic** — 한 노트에 하나의 아이디어만. 더 쪼갤 수 있으면 쪼개기.
2. **Own Words** — 인용 금지. 모든 노트는 내 언어로.
3. **Linking** — 새 노트 작성 시 반드시 기존 노트와 연결 ([[#개념ID]] 형식).
4. **Permanent** — 출처와 분리해도 그 자체로 이해 가능해야 함.

---

## 📋 Atomic Note 형식

```
### [#ID] 개념명
- **도메인**: D1~D8
- **태그**: #분야 #하위분야
- **한 줄 정수**: (1문장 — 전문용어 제거)

**내 말로** (Feynman 어린이 설명):
(여기에 풀어쓰기)

**왜 중요한가**:
(이 모델을 모르면 어떤 손해를 보는가)

**반례·한계**:
(이 모델이 안 통하는 경우)

**연결 (Linking)**:
- [[#ID-x]] (어떻게 연결되는지 한 줄)
- [[#ID-y]]
- [[#ID-z]]

**경영·일상 적용**:
- (구체 사례 1)
- (구체 사례 2)

**파생 질문**:
- (이 개념에서 나오는 미해결 질문)

**메타**:
- 첫 학습일: YYYY-MM-DD
- 출처: (책·강연·세션)
- 복습: D+1 ☐ / D+3 ☐ / D+7 ☐ / D+14 ☐ / D+30 ☐ / D+90 ☐
```

## 🗂️ Domain 1 — 인지 아키텍처

### [#D1-001] Inversion (역전 사고)
- **도메인**: D1
- **태그**: #인지아키텍처 #멘탈모델 #의사결정
- **한 줄 정수**: 목표에 직접 달려가는 대신, 실패 경로를 먼저 열거해서 봉쇄한다

**내 말로** (Feynman 어린이 설명):
이루고자 하는 바에 더 가까워지기 위해, 그 과정에서 하면 안 되는 것들을 피하는 사고의 전환. "어떻게 성공할까?" 대신 "어떻게 반드시 실패할까?"를 먼저 묻는다.

**왜 중요한가**:
인간은 장기 성과에 불리한 존재다 — 욕구와 즉각적 보상에 끌리기 때문에, 목표로 가는 직선 경로보다 이탈 경로를 먼저 봉쇄하는 게 현실적이다. 또한 인간은 이득보다 손실에 2배 민감하므로(Loss Aversion), "잃지 않을 것들"을 정의할 때 실행력이 더 강해진다.

**반례·한계**:
- 실패 경로 열거가 지나치면 행동 자체를 마비시킬 수 있다 (분석 마비)
- 전례 없는 혁신 상황에서는 알려진 실패 경로가 없어 Inversion만으론 부족하다

**연결 (Linking)**:
- [[#D1-002]] First Principles — 둘 다 전제를 해체하는 접근. Inversion은 목표를, FP는 문제를 해체
- [[#D1-003]] 확증편향 — Inversion이 막아야 할 대표적 실패 경로 (학습 예정 Week 3)

**경영·일상 적용**:
- 주식: "어떻게 잃을까?" → 전액 한 종목 투입 / 확증편향으로 버티기 / 시장 역행 고집 → 이 세 가지를 안 하면 된다
- 경영: "어떻게 조직을 망칠까?" → 소통 차단, 책임 회피, 단기 성과 집착 → 봉쇄 목록이 곧 리더십 원칙

**파생 질문**:
- Inversion으로 뽑은 실패 목록이 서로 충돌할 때는 어떻게 우선순위를 정하나?
- 실패 경로를 모르는 완전히 새로운 영역에서는 어떤 도구를 써야 하나?

**메타**:
- 첫 학습일: 2026-04-29
- 출처: Sapiens 세션 1 / Munger, C. *Poor Charlie's Almanack*
- 복습: D+1 ☐ / D+3 ☐ / D+7 ☐ / D+14 ☐ / D+30 ☐ / D+90 ☐
